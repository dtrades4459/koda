## 1 · Focus design

Implemented globally in `src/index.css`:

- **Ring:** 2 px solid `oklch(0.84 0.14 175)` (live mint) with a 22%-opacity
  4 px soft glow. Applied via `:focus-visible` only — mouse clicks don't show it.
- **Inputs (underline style):** swap ring for bold mint underline so the field
  doesn't visually balloon.
- **Links in flow text:** ring + underline; ensures keyboard users see both the
  hover and the focus state.
- **Skip link:** `.koda-skip-to-content` — invisible until focused, jumps to the
  main `<main id="main">` region. First focusable element on every page.

The mint ring keeps a contrast ratio of ≥ 3:1 against both the warm-dark surface
(`#13110E`) and the lighter panel surface (`#1F1C16`). Confirmed against WCAG 2.2
1.4.11.

## 2 · Keyboard-only navigation flows

### Global shortcuts
| Key | Action |
|---|---|
| `⌘K` / `Ctrl+K` | Open command palette |
| `?` | Toggle keyboard shortcuts overlay |
| `Esc` | Close topmost modal / sheet / drawer |
| `Tab` / `Shift+Tab` | Cycle focus |
| `Enter` / `Space` | Activate focused control |

### Tab order — per surface

Each surface lists tab order from top of the visible region.

**Auth (sign-in / sign-up)**
1. Skip-to-content link
2. Email field
3. Password field
4. Show/hide password toggle
5. Primary CTA (Sign in / Create account)
6. OAuth: Google → X → Apple
7. Forgot password? link
8. Toggle (sign-up / sign-in)

**Home (logged-in)**
1. Skip-to-content
2. Bell (notifications)
3. Avatar
4. Subnav tabs (Feed / Stats / Rules / Journal)
5. Primary content (varies by section)
6. Bottom tab bar (Home / Log / Stats / Circles / Social)

**Log trade**
1. Pair field
2. Side toggle (Long / Short)
3. Date / Time
4. Strategy picker
5. Outcome (Win / Loss / BE)
6. PnL field
7. Risk / Reward
8. Notes textarea
9. Screenshot upload button
10. Tag inputs
11. Save → Cancel

**Trade detail**
1. Back button
2. Edit button
3. Delete button
4. Emotion chips
5. Rule pills
6. Reaction buttons
7. Comment textarea
8. Send comment button

**Settings hub** — pure list, top to bottom, item-by-item.

**Modal / sheet rules**
- Focus moves into the modal on open; trapped within until close.
- First focusable element receives focus on open.
- Restore focus to trigger element on close.
- `Esc` always dismisses.

### Focus traps

Implementation pattern (used by `SessionExpiredModal`, `PermissionPrimerSheet`,
`SharedContentSheet`):

```tsx
useEffect(() => {
  const first = ref.current?.querySelector<HTMLElement>(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );
  first?.focus();
}, []);
```

For nested modals, only the topmost holds focus. The lower modal becomes
`inert` (or `aria-hidden="true"` + pointer-events none).

## 3 · Screen-reader announcement scripts

> Use `role="status"` + `aria-live="polite"` for non-blocking updates.
> Use `role="alert"` + `aria-live="assertive"` for critical errors only —
> assertive interrupts the user mid-sentence and should be rare.

### Toasts
- Success / info → `aria-live="polite"`. Body text reads verbatim.
- Warn → `aria-live="polite"`. Prefix with "Heads up: ".
- Error → `aria-live="assertive"`. Prefix with "Error: ".

### Banners (top)
- OfflineBanner — `role="status"` `aria-live="polite"`: "You're offline. Showing your last synced data. Changes save locally."
- ReconnectedBanner — `role="status"` `aria-live="polite"`: "Back online. Synced {n} trades logged offline."
- SlowConnectionBanner — `role="status"` `aria-live="polite"`: "Slow connection. Still loading."
- VersionMismatchBanner — `role="status"`: "Kōda was updated. Reload to avoid errors."

### Modals
- SessionExpiredModal — `role="dialog"` `aria-modal="true"` `aria-labelledby="..."`. Reads: "Session expired. For your security we signed you out after inactivity. Sign back in — your draft trade is saved."
- RateLimitedModal — same. Reads countdown updates as `aria-live="polite"`.
- Confirmation modals (delete, leave Circle, disconnect broker) — `role="alertdialog"` `aria-describedby="..."`. Confirm button labeled with verb + noun ("Yes, delete trade").

### Live regions
- Bell badge unread count — `aria-label="{n} unread notifications"`.
- Tilt-signal monitor — `aria-live="polite"` so signals don't barge in mid-trade.
- Pre-session checklist — fires "Rule checked: {label}" on each tick.
- Trade saved — `role="status"`: "Trade saved." (also via toast).
- Intervention cooldown — `aria-live="assertive"`: "Cooldown active. Kōda paused you for {duration}. Take a breath."

### Form errors
- Field-level: `aria-invalid="true"` + `aria-describedby` pointing to error text.
- Top-of-form summary on submit: focus jumps to summary `role="alert"`.
- Validation copy from [`microcopy.md §2`](microcopy.md).

### Charts & visualisations
- Each chart has a sibling `<table>` with `class="koda-sr-only"` containing the
  underlying data points. Screen readers get the numbers; sighted users see
  the chart.
- Sparklines have an `aria-label` summary: "Equity curve: starts at +0R,
  peaks at +24R, ends at +18R over 32 trades."

## 4 · Font scaling at 200%

- All typographic styles use absolute px values for visual control, but the
  browser zoom (`Ctrl/Cmd +`) scales them proportionally — including the
  `clamp()` functions used across headlines.
- Tap targets are min 44 × 44 px at 1× zoom. At 200% they're 88 × 88 — well
  above the 44 px WCAG 2.5.5 minimum.
- Line-height is unitless (1.4–1.65) so it scales with the font size.
- Containers use `max-width: 480–720px` with `padding: clamp(...)`, so text
  reflows at zoom instead of overflowing.

**Smoke test**: open the home screen, hit `Ctrl + +` until 200%. Headlines
should remain readable, no horizontal scroll on desktop > 720px, tap targets
remain hittable. Done.

## 5 · High-contrast mode

Implemented via `@media (prefers-contrast: more)` in `src/index.css`:

- Focus ring widens from 2 px → 3 px.
- Focus-ring offset 2 px → 3 px.
- Focus glow halo widens 4 px → 6 px and the mint shifts from 22% opacity → 40%.

We don't currently invert the colour palette — the warm-dark Kōda OS aesthetic
is intentionally low-contrast in places (e.g. `text2` against `bg`). To support
true high-contrast users we should add a second pass:

- Override `text2`, `text3`, `muted` to push contrast ≥ 7:1 against bg in
  `prefers-contrast: more`.
- Replace `color-mix()` softs (e.g. `liveSoft`) with full saturation.

**Status**: focus-state contrast satisfies WCAG 1.4.11 globally. Palette
contrast for plain text is on the Pro polish backlog.

## 6 · Reduced motion

Implemented globally in `src/index.css`:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

- All keyframes (kRise, kSlideIn, kRipple, kShimmer, kConfetti, etc.) are
  effectively stripped.
- Pull-to-refresh and swipe gestures still report success but the visual
  travel is removed.
- Page transitions: the fade-in stagger is bypassed.

## 7 · ARIA cheat-sheet (Kōda conventions)

| Element | role / aria | Example |
|---|---|---|
| Modal | `role="dialog"` `aria-modal="true"` `aria-labelledby` | SessionExpiredModal |
| Confirm dialog | `role="alertdialog"` | Delete trade |
| Banner | `role="status"` `aria-live="polite"` | OfflineBanner |
| Critical error | `role="alert"` `aria-live="assertive"` | Session expired toast |
| Tab list | `role="tablist"` with `role="tab"` + `aria-selected` | Stats subnav |
| Switch (theme picker) | `role="switch"` `aria-checked` | Settings |
| Disclosure | `aria-expanded` `aria-controls` | Subnav dropdowns |
| Live region (count) | `aria-label="{n} unread notifications"` | Bell badge |
| Decorative image | `aria-hidden="true"` | Background orbs |
| Icon button | `aria-label="{action}"` | All bare icon buttons |
